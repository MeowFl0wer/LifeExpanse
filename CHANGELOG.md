# 更新日志

本文件记录 LifeExpanse 每个版本的变更。

格式说明：每个版本按 **新增 / 变更 / 修复 / 已知限制** 四类记录。

版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)：`主版本.次版本.修订号`。
当前处于 `0.x` 阶段，接口和数据结构仍可能变动。约定：

- **只修 bug** → 递增修订号（`v0.5.0` → `v0.5.1`）
- **新增功能** → 递增次版本号（`v0.5.1` → `v0.6.0`），修订号归零
- 破坏性变更在 `0.x` 阶段也走次版本号，并在「变更」中标注

## [0.15.0] - 2026-07-20

文件与内容关联，以及缩略图。

### 新增

- **文件必定属于某条内容**。关联从正文和封面**自动推导**，
  客户端不需要也无法单独声明——独立的「附加」调用意味着任何忘记调用的
  路径都会静默产生孤儿，而忘记一定会发生（ADR-018）
- **孤儿清理**：未关联的上传保留 24 小时，被移出正文的保留 7 天。
  启动时自动执行，管理后台可手动触发并查看待清理数量
- **缩略图**：上传时生成 WebP（最长边 720px）。
  渲染内容时用缩略图，原图只在点击「查看原图」或「下载」时请求（ADR-019）
- 图片右下角的「查看原图」和「下载」，**悬停才出现**；
  触屏没有悬停，所以在这类设备上常驻
- 数据库元数据命名约定。SQLite 无法原地修改约束，
  Alembic 必须能给每个约束命名才能重写表

### 变更

- 内容进回收站时，其中的图片标记为孤儿而**不删除**——
  内容可能被恢复，恢复出来图片却没了是更糟的结果
- 内容彻底删除时，图片一并删除
- 删除文件时缩略图一并删除
- 图片单文件上限 10 MB（本来就是默认值，这次端到端验证过）

### 修复

- **新增列的迁移在有数据的表上会失败**：`has_thumbnail` 是 NOT NULL
  却没有默认值，已有行无法填充。用带真实数据的库验证后加上了 `server_default`
- 同一个迁移还会因为外键没有名字而报 `Constraint must have a name`

### 已知限制

- 视频没有封面帧（需要 ffmpeg）
- 缩略图不针对动图保留动画，点开原图才会动
## [0.14.0] - 2026-07-20

文件上传。上传权限位终于有了真正管得着的东西。

### 新增

- **图片与视频上传**。字节存磁盘（`LIFE_MEDIA_ROOT`），数据库只存元数据
- **类型由字节决定**，不看 `Content-Type` 也不看扩展名。
  只认 JPEG / PNG / GIF / WebP / MP4 / WebM
- **每个文件有自己的可见性，默认私密**。读取接口检查权限；
  不存在、已删除、无权限返回同一个 404
- **每用户存储配额**，删除后立即释放
- 单文件大小上限：图片 10 MB、视频 200 MB（可配置）
- `GET /media/quota` 返回已用空间与本账号的上传权限，
  编辑器据此在选文件之前就说明能不能传
- 管理后台的图片 / 视频计数接上真实数据（仍然只有数量和字节数，没有文件名）

### 变更

- 编辑器的插入媒体、文件夹与系列封面改为**真实上传**。
  原来用 `URL.createObjectURL`，刷新就失效
- 头像上传不受管理员授予的图片权限限制——头像是「有个账号」的一部分，
  不是内容特权；且上传即公开，否则访客看不到
- 删除是数据库软删 + **磁盘硬删**：用户要求删掉后还留着字节，
  是保留了错误的那一半

### 安全

- 返回时带 `X-Content-Type-Options: nosniff` 与 `Content-Disposition: inline`
- 磁盘路径由服务端随机 id 生成，客户端文件名只用于展示，
  `../../../../etc/passwd` 这类文件名无处可去（ADR-016）

### 已知限制

- 上传的文件目前不与具体内容关联，删除内容不会连带删除其中的图片
- 没有图片压缩或缩略图
- 备份与恢复仍是原型
## [0.13.0] - 2026-07-20

数据库迁移，以及备用邮箱。

### 新增

- **Alembic 迁移**取代 `create_all`。启动时自动 `upgrade head`，
  可重复运行。`upgrade` / `downgrade` / 应用内启动三条路径都验证过
- **备用邮箱**：绑定、解绑，**可用于登录和找回密码**。
  绑定与解绑需要当前密码 + 第二重证明（邮箱验证码或两步验证码），
  与更换主邮箱同一标准——多一个地址就是多一道门
- 绑定备用邮箱后通知主邮箱，通知里新地址是打码的

### 变更

- 应用内头部的站点标识点击回主页（原来指向 `/app`，在工作台上等于没反应）
- 备用邮箱必须**通过验证**才能登录：直接写进数据库的地址不被接受
- 主邮箱不能改成已经绑定为备用的地址
- 一个邮箱仍然只能属于一个账号，主备两栏一起算

### 修复

- **迁移会把整个进程的日志关掉**：Alembic 的 `fileConfig` 默认
  `disable_existing_loggers=True`，在应用内跑迁移等于启动时禁用
  uvicorn 和应用的全部日志——连上个版本刚修好的验证码输出也一起没了。
  这是端到端验证抓到的，158 个测试全绿也照样存在（ADR-015）

### 已知限制

- 文件上传仍未实现
- 备份与恢复仍是原型
## [0.12.0] - 2026-07-20

管理后台。账号与用户管理这一轮到此完成。

### 新增

- **管理后台**（`/admin`），六个分页：
  - **系统状况**：运行时间、负载、CPU、在线用户、活跃会话、内容总数
  - **用户**：按注册时间 / 最近登录 / 登录次数排序，可搜索。
    详情页展示基本信息、注册与登录时间、**内容数量统计**和权限开关
  - **注册与邀请**：三种注册模式实时切换；邀请码生成、查看、撤销
  - **审计日志**：按事件筛选，最新在前
  - **管理员账号**：管理员自己的密码与两步验证
  - **备份与恢复**：原有原型面板，原样保留
- 上传权限（图片 / 视频）与账号启用状态由管理员授予，**每次改动写入审计日志**
- 系统状况区分「在线用户」（15 分钟内建立过会话）与「活跃会话」（尚未过期的全部会话）

### 变更

- **管理员判定改为看服务端返回的角色**，不再比较用户名。
  站点主人 `euan` 是普通账号，管理员是独立账号（需求 3.1）
- 普通用户访问后台接口返回 **404 而不是 403**——403 会确认后台存在
- **管理后台不读取用户内容**：用户详情只返回数量，接口本身不返回标题或正文
- 用户列表与详情中的邮箱一律打码

### 修复

- **控制台邮件后端的验证码从未真正出现在任何地方**：
  邮件以 `INFO` 记录，但没有任何地方配置日志处理器，记录被直接丢弃。
  pytest 因为自带 handler 才看得到，真实部署下开发者永远收不到验证码，
  「不需要邮件服务商也能跑通」这个说法并不成立。
  现在启动时配置日志，并把控制台邮件提到 `WARNING`

### 已知限制

- 文件上传尚未实现，用户详情里的图片 / 视频计数恒为 0（权限位可提前授予）
- 备份与恢复仍是原型，需要独立的后端支持
- 备用邮箱的绑定界面尚未做
## [0.11.0] - 2026-07-20

两步验证，以及用户真正会碰到的那些账号页面。

### 新增

- **两步验证（TOTP）**：接标准 Authenticator。开启是三步——
  取密钥、扫码、**输入一个有效验证码才真正开启**。
  扫错二维码就开启的话，用户会在下次登出时被锁在自己门外
- **恢复码**：开启时一次性给出 10 个，哈希存储，之后再也无法查看。
  每个只能用一次，用完可重新生成（旧的立即作废）
- **登录时的两步验证挑战**：密码正确但需要第二因素时，
  服务端返回 401 且不建立会话——「需要 2FA」本身不是一次登录
- **AdminEuan 不允许关闭两步验证**
- **注册页接上真实流程**：两步式，先验邮箱再建号
- **找回密码页**（`/forgot-password`），登录页的「忘记密码」不再是弹窗占位
- **设置页安全区**：改密码、换主邮箱、两步验证开关，
  三处都要求当前密码 + 第二重证明
- `src/lib/reserved.ts`：保留用户名清单的前端副本，
  让注册表单在请求发出前就能给出提示

### 修复

- **AdminEuan 根本无法登录**：登录时把输入统一转成小写去匹配用户名，
  而管理员用户名带大写，永远匹配不上。现在两侧都不区分大小写
- `SecuritySettings` 里的验证码输入框没有和 label 关联，
  屏幕阅读器读不出用途

### 已知限制

- 管理后台（系统状况、用户列表、授权、审计日志）在下个版本
- 备用邮箱的绑定界面尚未做
## [0.10.0] - 2026-07-20

账号系统的后端：邮箱验证、注册模式、角色与权限。**前端注册页尚未接入。**

### 新增

- **邮箱验证码**：注册、找回密码、改密码、改邮箱都走验证码。
  哈希后存库（验证码本质是临时密码）、10 分钟有效、单次使用、
  5 次猜测上限、每小时 5 条限流
- **邮件发送**抽象为一个函数，开发用 console 后端把邮件写进日志，
  整套流程不需要邮件服务商也能跑通和测试；Resend 之后接上即可
- **注册模式**：关闭 / 邀请码 / 公开，管理员可在运行时切换。
  邀请码单次使用
- **主邮箱 + 备用邮箱**，两者在全表唯一——一个邮箱只能属于一个账号
- **角色与上传权限**：`role`、`can_upload_image`（默认开）、
  `can_upload_video`（默认关），由管理员授予
- **AdminEuan 引导创建**：首次启动自动建号。没有配置密码时生成一个强密码
  并只打印一次——未配置绝不能等于弱管理员
- **保留用户名清单**（需求 §259），防止用户名遮挡平台路由
- **静态加密**（`app/crypto.py`）：TOTP 密钥用 AES-256-GCM 加密，
  验证码与恢复码用带密钥的哈希。范围与理由见 ADR-012
- **TOTP 与恢复码的底层实现**（`app/totp.py`），接口在下个版本

### 变更

- **改密码现在必须同时提供当前密码和第二重证明**（邮箱验证码或两步验证码）。
  只知道当前密码不够——没锁屏的电脑不该等于账号
- **改邮箱**需要当前密码 + 身份证明 + 新邮箱的验证码，三者齐备
- **邮箱变更后通知旧邮箱**，且新地址在通知里是打码的
- 重置密码会吊销全部已有会话
- 记录 `login_count` 与 `last_login_at`，供管理后台排序
- 设置页新增「返回我的」入口

### 修复

- **注册接口会泄露邮箱是否已注册**：邮箱重复时返回 409「该邮箱已被使用」，
  任何人都能用它把一份邮箱列表刷成「谁在这个站有账号」。
  现在所有陌生人可达的账号接口，存在与否响应完全一致（ADR-013）

### 已知限制

- 前端注册页、找回密码页、2FA 设置界面在下个版本
- 内容正文不在应用层加密，理由见 ADR-012（会废掉关键词搜索，
  且防不住被攻陷的服务器）；静态加密交给部署时的卷加密
## [0.9.0] - 2026-07-20

账号与用户管理改造的第一步：「我的」页面。

### 新增

- **`/me`「我的」页面**，登录后的账号中枢。顺序为
  个人信息 → 设置 → 记录概况 → 数据导出 → About
- 个人信息展示头像、昵称、用户名、签名和邮箱。
  **邮箱中间段打码**（`e***n@example.com`），且掩码宽度固定，
  不随地址长度变化——变宽的掩码会泄露长度本身
- 用户资料新增主邮箱与备用邮箱字段

### 变更

- 「记录概况」从 About 移到「我的」，存储用量一并并入
- 顶部导航最后一项按登录状态切换：登录后是「我的」，访客仍是「关于」。
  **About 没有移到登录之后**——它的作用正是向初次到访的人解释这个站是什么
- 应用内头像和菜单入口指向 `/me` 而不是直接进设置

### 已知限制

- 页面数据仍来自内存实现，改资料、导出等仍是入口占位
- 邮箱验证、注册模式、2FA、权限与管理后台在后续版本
## [0.8.0] - 2026-07-20

笔记与文章的筛选栏和阅读版式调整。

### 新增

- **文件夹和系列现在可以搜索**：以前一进 Folders / Series，关键词框就没反应了。
  现在按名称和简介筛选——和内容搜索覆盖标题与摘要保持一致，
  只搜名称会漏掉卡片上明明看得见的词。没有匹配时给出说明，而不是留一片空白
- **笔记也有目录了**，不再只有文章。长笔记和长文章一样需要大纲

### 变更

- 「选择标签」移到「草稿」和关键词搜索之间，与筛选按钮同一行，不再单独占一行
- 目录移到左侧，正文在右侧
- 目录的显示条件改为「正文里有没有标题」，而不是「是不是文章」。
  没有标题就不显示——否则会白占一列侧栏，正文旁边空一块
- 进入某个文件夹或系列后，搜索框提示词回到「关键词搜索」，
  因为此时筛的是里面的内容而不是索引

## [0.7.0] - 2026-07-20

补齐工程流程：CI 上线，并把开发过程中的关键取舍写成决策记录。

### 新增

- **CI**（`.github/workflows/ci.yml`）：每次 push 到 main 和每个 PR 自动跑
  前端类型检查 + 305 个测试 + 构建，后端 59 个测试。
  用 `--frozen-lockfile` 保证 CI 装的依赖和干净 clone 一致
- **`docs/DECISIONS.md`**：11 条架构决策记录，含背景、取舍、代价和已知欠账
- `src/api/comments.ts`：评论数据层，同样按 `usingBackend()` 分流
- 评论可以真正删除了：本人删自己的，作者删自己内容下的，删除需二次确认

### 修复

- **双向链接在 HTTP 模式下仍读本地内存**：详情页已从后端取文章，
  但 `getLinkGraph` 无视分流，导致后端创建的笔记不参与反链，
  本地示例内容反而会混进后端页面的链接图。
  现在候选集合从服务器取（已按访客过滤），解析仍留在前端——
  理由见 ADR-005：两个 Markdown 解析器一定会跑偏
- **评论前端没有接后端**：`CommentSection` 直接读写 mock store，
  所以后端评论权限规则修好了，真实页面发评论却不写服务器、也不跨会话保留；
  删除和隐藏还是 `alert` 原型。现在走数据层
- **文件夹/系列保存失败没有任何反馈**：`.then()` 没有配 `.catch()`，
  HTTP 失败时用户会停在编辑态且毫无提示。
  连带发现**新建路径也是同一个问题**——表单的 `onSubmit` 返回 `void`，
  拒绝会变成未处理的 Promise rejection。现在表单会等待并把失败显示在原地

### 变更

- 提交信息不再附带 AI 共同署名

### 已知限制

- 链接图会拉取作者的可见内容列表；内容量很大时需要改成后端出接口（ADR-005）
- 评论的「隐藏」（区别于删除）后端有字段但前端未做
## [0.6.0] - 2026-07-20

前端接入后端。设置 `VITE_API_BASE` 后，笔记与文章模块与登录会真正走服务器。

### 新增

- `src/api/pkmHttp.ts`：笔记与文章的 HTTP 实现，负责 snake_case ↔ camelCase 转换
- `src/api/auth.ts`：登录 / 登出 / 恢复会话；会话是 HttpOnly Cookie，
  身份从 `/auth/me` 取回而不是本地解码
- `src/api/pkm.ts` 改为分流：配了 `VITE_API_BASE` 走 HTTP，否则走内存实现。
  页面调用的函数名不变，**切换后端不需要改任何页面**

### 变更

- 版本号约定写入 CHANGELOG：只修 bug 递增修订号，新增功能递增次版本号

### 修复

- **仍有前端写路径绕过 actor API**：新建 PKM、编辑保存、详情页删除、
  文件夹与系列的创建和保存都在直写存储。现在全部经过数据层，
  「写接口是唯一入口」这条契约才真正成立
- **归档内容会回到默认列表**：列表页传了 `includeArchived: true`，
  与编辑页「归档（不再出现在默认列表中）」的说明冲突
- **后端未强制「只有 Article 可以开启评论」**（需求 10.1）：
  创建时传 `allow_comments: true` 即使是笔记也会保存，patch 也能改，
  评论接口只看开关不看形态。现在三处都按当前形态判定，
  文章退回笔记后会立即停止接受评论

### 已知限制

- 随想、日记、轨迹、足迹、飞行、加密空间尚未接入后端
- 前端仍以内存实现为默认；`VITE_API_BASE` 未设置时行为不变

## [0.5.0] - 2026-07-20

新增后端。项目从纯前端原型变为前后端两端俱全，可用 Docker Compose 部署。

### 新增

**后端（`backend/`，FastAPI + SQLAlchemy + SQLite）**
- 鉴权：注册、登录、登出、当前用户、设备会话列表。
  密码用 Argon2id 哈希，会话是不可猜测的随机 id，存 HttpOnly Cookie；
  勾选「保持登录」才设 `max_age`，否则随浏览器会话结束
- 内容：列表、按作者与类型取单条、创建、更新、发布为文章、退回笔记、删除
- 库：文件夹与系列的增删改查，多对多归属，系列继承规则在服务端执行
- 回收站：软删除、恢复（自动解决 slug 冲突）、彻底删除、清空、超期清理
- 草稿：服务端存储，**跨设备可用**
- 评论：仅登录用户可发表，作者可删除自己文章下的评论
- 55 个后端测试，其中权限相关 20 个

**部署**
- `backend/Dockerfile`、`Dockerfile.frontend`、`nginx.conf`、`docker-compose.yml`、`.env.example`
- 前端与后端同源部署，nginx 反代 `/api`，Cookie 保持 first-party
- 数据库挂在具名卷上，不随容器销毁

**前端**
- `src/api/http.ts`：HTTP 传输层。设置 `VITE_API_BASE` 即走后端，
  不设则继续用内存实现——这样后端落地不必和所有页面、测试同一天改完
- 草稿改为「本地 + 服务端」双写：本地副本保证断网不丢（需求 25.2），
  服务端副本实现跨设备；读取优先服务端

### 修复

- **数据层的权限过滤没有真正生效**：PKM 列表页与详情页仍直接读 mockData
  并在组件里自行过滤，`src/api/pkm.ts` 的权限测试是绿的但 UI 绕得过去。
  两页现在都通过数据层读取
- **`getPkmBySlug` 未按作者和类型收敛**：全局按 slug 取第一条，
  `/alice/pkm/<euan的slug>` 可能显示不属于该用户的内容，日记的 slug 也能从
  pkm 路由打开
- **写接口没有 actor**：数据层无法执行 owner-only 规则。十个写接口现在都需要
  actor 并校验归属，非本人一律 404
- **草稿 key 未按用户隔离**：同一浏览器上 euan 写到一半退出，
  alice 登录后会看到 euan 的草稿。key 加用户前缀，登出时清除该用户草稿
- **`useAutosave` 在卸载后仍可能 setState**：写入照常完成，状态更新加守卫

### 已知限制

- **前端页面尚未切到 HTTP**：传输层已就绪，但页面仍走内存实现。
  切换需要逐页处理 loading / error，是下一步
- 后端未实现：加密空间、人生轨迹 / 足迹 / 飞行、文件上传、数据导出与备份
- 用 `create_all` 建表，schema 变动前应换成 Alembic
- 生产环境需把会话 Cookie 的 `secure=True` 打开

  「公开」由绿色改为青色，避免与「已选中」的主色绿混淆
- 标签筛选由下拉框改为可展开的多选标签条，移除独立的 Tags 视图

- 登录会话：勾选「保持登录」才写入 `localStorage`，
  否则写入 `sessionStorage`，关闭标签页即失效

### 修复

- **随想与日记页面错误渲染为「笔记与文章」**：路由使用字面路径，
  `useParams().section` 恒为 `undefined` 并回落到 `pkm`；
  同一问题还导致详情页与编辑页的面包屑生成 `/euan/undefined`
- **登录状态在首页失效**：会话读取没有订阅机制，已挂载的头部
  始终渲染登出态；改为响应式会话存储
- **私密与草稿内容可通过 slug 直达**：列表与搜索已过滤，
  但详情页未校验，现返回与不存在内容相同的 404
- **访客可看到私密文件夹与系列的名称和说明**：索引页与下钻路径
  均改为从过滤后的集合解析
- **删除的内容仍显示在首页与工作台**：两页读取的是模块加载时的
  种子数组快照；改为统一从内容主存储派生，并将种子数组取消导出
- **新建的内容不会出现在首页与工作台**：与上一条同源
- **`/new/note`、`/account` 等受保护路由登录后落到 `/app`**：
  路由守卫未携带 `next` 参数
- **`?next=` 校验过松**：接受 `//evil.example` 等协议相对地址
- **「保持登录」复选框无实际作用**：无论是否勾选都持久化会话
- **足迹表单不合并同名城市**：现按「城市 + 国家」合并并累加到访次数
- **同名内容 slug 冲突**：两条标题相同的内容会得到同一 slug，
  保存后跳转到旧内容
- **批量轨迹的「写入地图」未真正写入**
- **CSV 导入按逗号切分**：引号内含逗号的字段会解析错误，
  改为 RFC 4180 解析
- **活跃热力图使用随机数**：刷新即变化，改为按日期哈希
- **登录成功后的跳转定时器未清理**：在 600ms 内离开页面会被拉回
- 加密空间锁定后界面不会自动解锁（无倒计时触发重渲染）
- 标签筛选下拉框为模块加载时快照，新建标签不会出现

### 已知限制

- **没有后端**。全部数据保存在浏览器内存，刷新页面即还原
- 账号资料保存、数据导出导入、系统备份恢复、空间新建与设置
  等操作仍为界面演示
- 权限判断在前端进行，仅用于原型演示，不具备真实安全性
- 文件夹与系列不支持多级嵌套（当前为系列 > 文件夹 > 内容两层）
- 文件夹与系列尚不支持删除
- 页面组件测试覆盖主要流程，地图绘制与编辑器交互仍需手动验证
